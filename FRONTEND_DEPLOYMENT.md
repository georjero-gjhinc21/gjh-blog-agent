## Deployment Options

### Option 1: Deploy to Vercel (Recommended)

#### Method A: Vercel CLI

```bash
# Install Vercel CLI
npm install -g vercel

# Deploy frontend
cd /opt/gjh-blog-agent/frontend
vercel

# Follow prompts:
# - Link to existing project? No
# - Project name: gjh-blog
# - Directory: ./
# - Override settings? No

# Deploy to production
vercel --prod
```

#### Method B: GitHub + Vercel Dashboard

1. **Push frontend to GitHub** (already done!)

2. **Go to Vercel Dashboard**:
   - Visit: https://vercel.com/new
   - Click "Import Project"
   - Select your GitHub repository: `gjh-blog-agent`

3. **Configure Project**:
   ```
   Project Name: gjh-blog
   Root Directory: frontend
   Build Command: npm run build
   Output Directory: .next
   Install Command: npm install
   ```

4. **Add Environment Variables**:
   ```
   NEXT_PUBLIC_SITE_URL=https://gjhconsulting.net
   NEXT_PUBLIC_SITE_NAME=GJH Consulting
   ```

5. **Deploy!**
   - Click "Deploy"
   - Wait ~2 minutes for build
   - Your site is live!

6. **Configure Custom Domain**:
   - Go to Project Settings → Domains
   - Add: `gjhconsulting.net`
   - Follow DNS instructions

### Option 2: Deploy to Netlify

```bash
# Install Netlify CLI
npm install -g netlify-cli

# Deploy
cd /opt/gjh-blog-agent/frontend
netlify deploy --prod

# Configure:
# - Build command: npm run build
# - Publish directory: .next
```

### Option 3: Self-Hosted (Docker)

Create `frontend/Dockerfile`:

```dockerfile
FROM node:18-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
RUN npm run build

FROM node:18-alpine
WORKDIR /app
COPY --from=builder /app/.next ./.next
COPY --from=builder /app/node_modules ./node_modules
COPY --from=builder /app/package.json ./package.json
EXPOSE 3000
CMD ["npm", "start"]
```

Then:

```bash
docker build -t gjh-blog-frontend ./frontend
docker run -p 3000:3000 gjh-blog-frontend
```

## Troubleshooting

### Build Fails

```bash
# Clear cache
rm -rf .next node_modules
npm install
npm run build
```

### Posts Not Showing

Check path in `lib/posts.ts`:

```typescript
const postsDirectory = path.join(process.cwd(), '../data/posts');
```

For Vercel, you may need to adjust this path or copy posts during build.

### Styling Issues

```bash
# Rebuild Tailwind
npm run build
```
