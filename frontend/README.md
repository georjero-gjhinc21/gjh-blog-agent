# GJH Consulting Blog - Next.js Frontend

Beautiful, SEO-optimized blog frontend that displays AI-generated content from the GJH Blog Agent.

## Features

✅ **Server-Side Rendering** - Fast, SEO-friendly pages
✅ **Responsive Design** - Beautiful on all devices
✅ **Tailwind CSS** - Modern, customizable styling
✅ **TypeScript** - Type-safe development
✅ **Markdown Support** - Renders blog posts from markdown files
✅ **SEO Optimized** - Meta tags, Open Graph, Twitter Cards
✅ **Affiliate Integration** - Ready for PartnerStack links
✅ **Vercel Ready** - One-click deployment

## Quick Start

### 1. Install Dependencies

```bash
npm install
```

### 2. Development Server

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) to view the site.

### 3. Build for Production

```bash
npm run build
npm start
```

## How It Works

### Blog Post Reading

The frontend reads markdown files from `../data/posts/` (generated by the Python backend):

1. **getAllPosts()** - Lists all blog posts
2. **getPostBySlug()** - Reads individual post
3. **markdownToHtml()** - Converts markdown to HTML

### SEO Optimization

Each blog post includes:
- Dynamic meta tags
- Open Graph tags
- Twitter Card tags
- Structured keywords
- Proper HTML semantics

### Styling

Using Tailwind CSS with custom configuration:
- Primary color: Blue (#0ea5e9)
- Typography plugin for blog content
- Responsive breakpoints
- Custom components (buttons, cards)

## Customization

### Change Colors

Edit `tailwind.config.ts`:

```typescript
colors: {
  primary: {
    // Your color palette
  }
}
```

### Update Content

Blog posts are automatically read from `../data/posts/`. Just generate new posts with the Python backend!

### Add Pages

Create new files in `app/`:

```
app/services/page.tsx
app/resources/page.tsx
```

## Integration with Backend

The frontend reads blog posts from the Python backend's output:

```
/opt/gjh-blog-agent/
├── data/posts/           ← Frontend reads from here
│   └── *.md
└── frontend/            ← Next.js app
```

Posts are generated by:
```bash
cd /opt/gjh-blog-agent
./venv/bin/python main.py workflow
```
